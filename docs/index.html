// Financial Risk Management System

RISCO
DBMS

A full-stack financial database management system for startups facing uncertainty — built with FastAPI, Oracle DB, and React.

View on GitHub ↗ Installation Guide

01 — Overview

What is RISCO?

Organizational Hierarchy

Manage financial data across a structured hierarchy: organizations → members → clients → revenue/expenses → risk alerts → snapshots.

Risk Intelligence

Automated risk alerts generated from financial thresholds, with planned predictive analysis using Base, Worst, and Best case financial snapshots.

Predictive Analysis

NumPy-powered financial snapshot calculations using client reliability scores, with ANN models via TensorFlow or Scikit-learn planned.

Clean Architecture

Monolithic 3-tier design with clean internal layering — routers → services → models — for high cohesion and low coupling.

04 — Database Schema

9 tables, normalized to 4NF

Users
System users with authentication info
Organization
Top-level organizational entities
Roles
Roles within an organization
OrganizationMembers
Users linked to organizations with roles
Clients
Clients belonging to an organization
Revenue
Revenue records linked to clients
Expenses
Expense records linked to organizations
RiskAlerts
Alerts generated from financial thresholds
FinancialSnapshots
Periodic summaries — Base / Worst / Best case

06 — Installation

Get it running locally

Prerequisites: Python 3.11, Conda, Oracle Database 21c (with a configured Pluggable Database), Node.js (for React frontend), GitHub CLI (optional).

01

Clone the repository

git clone https://github.com/AliRazaDev241/risco.git
cd risco
02

Create and activate the Conda environment

conda create -n risco python=3.11
conda activate risco
03

Install dependencies

pip install -r requirements.txt
04

Configure environment variables

cp .env.example .env

Open .env and fill in your Oracle credentials (see below).

05

Initialize the database

python init_db.py

Creates all tables in your Oracle PDB. Safe to re-run — only creates missing tables.

06

Start the development server

uvicorn main:app --reload

API available at http://localhost:8000 — Swagger UI at /docs

Environment Variables

DB_HOST=localhost
DB_PORT=1521
DB_SERVICE_NAME=orclpdb.bbrouter   # verify with: lsnrctl status
DB_USER=risco
DB_PASSWORD=your_password_here

⚠ Never commit your .env file — it is already listed in .gitignore.
